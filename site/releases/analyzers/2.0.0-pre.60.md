Today, we're shipping two new prereleases:

* xUnit.net Core Framework v3 `4.0.0-pre.154` ([release notes](/releases/v3/4.0.0-pre.154))
* **xUnit.net Analyzers `2.0.0-pre.56`**
* xUnit.net Visual Studio adapter `4.0.0-pre.5` ([release notes](/releases/visualstudio/4.0.0-pre.5))

This is the fifth prerelease build for the 2.0 release.

As always, we'd like to thank all the users who contributed to the success of xUnit.net through usage, feedback, and code. 🎉

## Release Notes

These release notes are a list of changes from `2.0.0-pre.56` to `2.0.0-pre.60`.

### Usage Analyzers

* We have updated [xUnit1010](/xunit.analyzers/rules/xUnit1010) to stop reporting string values which are correctly converted at runtime to their target numeric type (for example, `"123"` passed for an `int`). [xunit/xunit#2354](https://github.com/xunit/xunit/issues/2354){: .issue-link }

* We have updated [xUnit1025](/xunit.analyzers/rules/xUnit1025) to report duplicate inline data where the values are the same but come from different types (for example, `1` as an `int` vs. `1` as a `long`). [xunit/xunit#2957](https://github.com/xunit/xunit/issues/2957){: .issue-link }

### Assertion Analyzers

* We have added [xUnit2033](/xunit.analyzers/rules/xUnit2033) which will alert users when they are failing to take advantage of assertions which return values (like `Assert.Single` and `Assert.IsType`).

* **BUG:** We have fixed an issue with the fixer for [xUnit2023](/xunit.analyzers/rules/xUnit2023) which was incorrectly duplicating comments and/or not correctly formatting the resulting changed code. [xunit/xunit#3336](https://github.com/xunit/xunit/issues/3336){: .issue-link }
